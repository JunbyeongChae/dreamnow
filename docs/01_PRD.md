## 0. 프로젝트 개요

이 문서는 **배익거리(配益居里) 기업 홈페이지**의 제품 요구사항을 정의한다.
오프라인 매장 중심으로 운영되어 온 베이커리카페 배익거리는 지점 확장과 함께 브랜드를 신뢰감 있게 소개하고, 메뉴 정보와 공지·문의 대응 채널을 제공할 공식 웹사이트가 필요한 상황이다. 이 서비스는 방문객에게는 기업 소개·메뉴 정보·고객센터를, 관리자에게는 개발자 도움 없이 배너·팝업·메뉴·상담 답변을 각 화면에서 직접 운영할 수 있는 관리 기능을 제공한다.
이 PRD를 기반으로 기능 구현 및 기술 결정이 이루어진다.

## 1. 배경 및 목표

### 1-1. 배경

배익거리는 인스타그램 등 SNS로만 매장 소식과 메뉴를 전달하고 있어, 검색 노출이 되지 않고 브랜드에 대한 신뢰도를 확보하기 어렵다. 지점이 늘어날 경우 공지사항(휴무, 신메뉴, 채용 등)과 고객 문의를 한 곳에서 관리하고 응대할 창구의 필요성이 커질것으로 예측된다. 또한 마케팅 담당자가 개발 지식 없이도 배너·팝업·메뉴를 직접 등록·수정하고 고객 문의에 답변할 수 있어야 운영 부담이 줄어든다.

### 1-2. 목표

| # | 목표 | 연결 성공 기준 (→ 섹션 2) |
|---|---|---|
| G-01 | 방문자가 브랜드 소개와 메뉴 정보를 3단계 이내 클릭으로 확인할 수 있다 | 핵심 기능 도달 단계 수 |
| G-02 | 관리자가 개발자 개입 없이 배너·팝업·메뉴·공지사항·상담 답변을 각 화면에서 직접 처리할 수 있다 | 관리자 권한별 기능 정상 동작 여부 |

### 1-3. 비목표

- 온라인 주문/결제 기능 — 이번 과제 범위 밖. 매장 방문 유도가 목적이며 커머스 기능은 별도 프로젝트로 고려
- 소셜 로그인(카카오, 네이버 등) — 아이디/비밀번호 기반 인증만 구현하기로 범위를 확정
- 다국어 지원 — MVP는 한국어만 지원하며, 확장 시 i18n 도입 고려

## 2. 성공 기준

### 정량 기준

| 지표 | 목표치 | 측정 시점 | 연결 목표 (← 섹션 1-2) |
|---|---|---|---|
| 메인 → 메뉴 상세 도달 단계 수 | 3단계 이내 | 구현 완료 후 직접 테스트 | G-01 |
| 관리자 배너/팝업 CRUD, 메뉴 등록, 공지사항 등록, 상담 답변 정상 동작 | 전체 동작 | 구현 완료 후 QA | G-02 |

### 출시 전 체크리스트

- [ ] 방문자가 배너/팝업/공지사항을 통해 최신 소식을 확인할 수 있다
- [ ] 방문자가 메뉴소개에서 카테고리별 메뉴 카드와 상세 정보(이미지·이름·가격·설명)를 확인할 수 있다
- [ ] 관리자가 메인페이지에서 배너·팝업을 등록·수정·삭제할 수 있다
- [ ] 관리자가 메뉴소개 화면에서 신규 메뉴를 등록할 수 있다
- [ ] 관리자가 고객센터에서 공지사항을 등록할 수 있다
- [ ] 관리자가 고객센터에서 모든 상담글을 상세 조회하고 답변을 등록할 수 있다
- [ ] 빈 상태·에러 상태에서 안내 메시지가 표시된다
- [ ] 모바일 375px 이상에서 레이아웃이 깨지지 않는다
- [ ] 주요 브라우저(Chrome, Safari)에서 정상 동작한다

## 3. 대상 사용자

### Primary — 매장 방문 관심 고객

| 항목 | 내용 |
|---|---|
| 대표 유형 | 배익거리 매장을 이용했거나 관심 있는 20~40대 |
| 주요 특징 | 모바일로 브랜드 소식과 메뉴 정보를 확인, SNS와 검색을 함께 사용 |
| Pain Point | 공식 채널이 없어 최신 공지(휴무, 신메뉴)와 메뉴 정보를 놓치기 쉽다 |
| 핵심 목표 | 브랜드와 메뉴를 빠르게 확인하고 방문 여부를 결정하고 싶다, 궁금한 점은 문의로 해결하고 싶다 |

### Secondary — 마케팅 운영 담당자(관리자)

| 항목 | 내용 |
|---|---|
| 대표 유형 | 배익거리 본사 마케팅 담당자, 비개발자 |
| 주요 특징 | 배너·팝업·메뉴·공지사항을 수시로 등록해야 하고, 고객 문의에 직접 응대해야 함 |
| Pain Point | 개발자 도움 없이는 콘텐츠 하나 바꾸기 어렵고, 별도 관리자 화면 없이는 문의 응대가 지연됨 |
| 핵심 목표 | 각 화면에서 직접 배너·팝업·메뉴·공지사항을 등록하고, 고객 문의에 신속히 답변하고 싶다 |

> 관리자는 데스크톱(또는 태블릿) 환경에서만 로그인해 운영하는 것으로 가정한다. 모바일에서는 관리자 전용 등록/답변 UI를 노출하지 않으며, 모바일 반응형 요구사항(5-1, 4장 각 화면의 "반응형" 항목)은 일반 방문자 기준으로만 적용된다.

## 4. 기능 요구사항

### 4-1. 메인페이지 (`/`)

| 기능 | 세부 요구사항 | 관련 스토리 |
|---|---|---|
| 배너 노출 | 등록된 배너를 노출 기간·순서에 따라 슬라이드로 표시 | US-01 |
| 팝업 노출 | 접속 시 활성화된 팝업을 노출, "오늘 하루 보지 않기" 옵션 제공 | US-02 |
| 퀵 링크 | 기업소개·메뉴소개·고객센터로 바로 이동하는 메뉴 카드 노출 | - |
| (관리자) 배너/팝업 관리 | 관리자가 로그인 후 헤더의 관리자 모드를 켜면 메인페이지 내에서 배너·팝업을 등록·수정·삭제(노출 기간·순서 포함) 가능. 별도 관리자 페이지 없이 인라인으로 처리 | US-03 |

**엣지 케이스**

- **빈 상태**: 활성 배너가 없으면 배너 영역 자체를 숨긴다
- **에러 상태**: 데이터 조회 실패 시 "잠시 후 다시 시도해주세요" 메시지 표시
- **로딩 상태**: 배너 이미지 로딩 중 스켈레톤 UI 표시
- **반응형**: 모바일은 배너 1장씩 풀폭 슬라이드 + 퀵 링크 2열 그리드, 태블릿은 배너 풀폭 유지 + 퀵 링크 3열, 데스크톱은 배너 고정 비율 + 퀵 링크 3~4열 가로 배치(브레이크포인트는 5장 참고)

### 4-2. 로그인/회원가입 (`/login`, `/signup`)

| 기능 | 세부 요구사항 | 관련 스토리 |
|---|---|---|
| 회원가입 | 아이디/비밀번호 기반, 비밀번호 규칙 검증 | US-04 |
| 로그인 | JWT 발급, role(user/admin) 구분 | US-05 |
| 로그인 유도(가드) | 비로그인 사용자가 인증이 필요한 액션(예: 고객센터 상담글 작성)을 시도하면 로그인 화면으로 이동, 로그인 화면에서 회원가입 화면으로도 이동 가능 | US-17 |

**엣지 케이스**

- **빈 상태**: 해당 없음
- **에러 상태**: 아이디/비밀번호 불일치 시 명확한 에러 메시지, 중복 아이디 가입 시도 시 안내
- **로딩 상태**: 로그인 처리 중 버튼 비활성화
- **반응형**: 모바일은 폼 영역이 좌우 여백만 두고 화면 폭 전체를 사용, 태블릿·데스크톱은 폼 카드를 최대 400px 폭으로 고정해 화면 중앙에 정렬

### 4-3. 기업소개 (`/about`)

| 기능 | 세부 요구사항 | 관련 스토리 |
|---|---|---|
| 브랜드 스토리 | 브랜드 소개, 연혁, 철학 등 콘텐츠 노출 | US-06 |
| 매장 안내 | 지점명·주소·영업시간 목록 노출 | US-06 |

**엣지 케이스**

- **빈 상태**: 해당 없음 (정적 콘텐츠)
- **에러 상태**: 콘텐츠 로드 실패 시 "잠시 후 다시 시도해주세요" 메시지 표시
- **로딩 상태**: 콘텐츠 로딩 중 스켈레톤 UI 표시
- **반응형**: 모바일은 브랜드 스토리 이미지·텍스트가 세로로 쌓이고 매장 안내가 1열 카드 목록, 태블릿은 매장 안내 2열, 데스크톱은 이미지·텍스트를 좌우로 배치하고 매장 안내를 2~3열 그리드로 노출

### 4-4. 메뉴소개 (`/menu`)

| 기능 | 세부 요구사항 | 관련 스토리 |
|---|---|---|
| 카테고리 사이드바 | 시즌메뉴, 음료, 디저트를 사이드바 상위 카테고리로 노출. 음료는 하위 카테고리(coffee, non-coffee, tea, ade/juice)를 펼쳐서 노출하며, 시즌메뉴·디저트는 바로, 음료는 하위 카테고리 선택 시 해당 화면으로 리렌더링 | US-07 |
| 메뉴 카드 목록 | 선택된 카테고리(또는 음료의 하위 카테고리)에 속한 메뉴를 카드(이미지 + 이름만) 형태로 목록 노출 | US-07 |
| 메뉴 상세 조회 | 카드 클릭 시 이미지·이름·가격·설명이 포함된 상세 화면 노출 | US-08 |
| (관리자) 메뉴 등록 | 관리자가 로그인 후 헤더의 관리자 모드를 켜면 메뉴소개 화면에서 상위 카테고리(시즌메뉴/음료/디저트)를 지정하고, 음료 선택 시 하위 카테고리(coffee/non-coffee/tea/ade·주스)까지 지정하여 신규 메뉴(이미지·이름·가격·설명)를 등록 가능. 별도 관리자 페이지 없이 인라인으로 처리 | US-09 |

**엣지 케이스**

- **빈 상태**: 선택한 카테고리에 메뉴가 없으면 "등록된 메뉴가 없습니다" 안내
- **에러 상태**: 메뉴 데이터 조회 실패 시 "잠시 후 다시 시도해주세요" 메시지 표시
- **로딩 상태**: 메뉴 카드/상세 이미지 로딩 중 스켈레톤 UI 표시
- **반응형**: 모바일은 좌측 사이드바가 상단 가로 스크롤 탭(또는 드롭다운)으로 전환되고 메뉴 카드 2열 그리드, 태블릿은 사이드바 고정 노출 + 카드 3열, 데스크톱은 사이드바 고정 노출 + 카드 4열

### 4-5. 고객센터 (`/support`)

| 기능 | 세부 요구사항 | 관련 스토리 |
|---|---|---|
| 공지사항 목록/상세 조회 | 휴무·신메뉴·채용 등 공지사항을 최신순으로 노출, 클릭 시 상세 조회 | US-10 |
| 1:1 문의(상담) 작성 | 로그인한 사용자가 상담글을 작성 | US-11 |
| 문의 목록 조회 | 일반 사용자는 본인이 작성한 문의만 조회 가능 | US-12 |
| (관리자) 상담글 상세 조회 및 답변 | 관리자가 로그인 후 헤더의 관리자 모드를 켜면 모든 사용자의 상담글을 예외 없이 상세 조회하고 답변을 등록할 수 있음 | US-13 |
| (관리자) 공지사항 등록 | 관리자가 로그인 후 헤더의 관리자 모드를 켜면 고객센터 화면에서 공지사항(제목·본문)을 직접 등록 가능. 별도 관리자 페이지 없이 인라인으로 처리 | US-14 |
| 공지사항 댓글 | 로그인한 사용자는 공지사항 상세에 댓글을 작성할 수 있고, 비로그인 사용자는 조회만 가능. 작성자 본인 또는 관리자만 댓글 수정·삭제 가능 | US-19 |
| (관리자) 공지사항 AI 초안 생성 | 관리자가 공지사항 작성 시 제목을 입력하면 Gemini API로 본문 초안을 생성해 에디터에 채워주는 버튼 제공. 생성된 초안은 관리자가 검토·수정 후 등록 | US-20 |

**엣지 케이스**

- **빈 상태**: 공지사항/문의 내역이 없으면 "등록된 공지사항(문의)이 없습니다" 안내, 댓글이 없으면 "댓글 0"으로 표시
- **에러 상태**: 일반 사용자가 타인의 문의 상세에 접근 시 403 처리, 목록에서 답변 상태(답변대기/답변완료) 뱃지로 구분. AI 초안 생성 실패(API 키 미설정, 외부 API 오류) 시 에러 메시지 노출하고 본문은 비워둔 채 관리자가 직접 작성 가능
- **로딩 상태**: 목록 조회 중 스켈레톤 표시, AI 초안 생성 중 버튼에 "생성 중..." 표시
- **반응형**: 모바일은 목록이 풀폭 리스트로 노출되고 항목 클릭 시 상세 화면으로 풀스크린 전환, 태블릿·데스크톱은 목록이 중앙 정렬된 고정폭 컨테이너 안에 표시되고 상세는 같은 컨테이너 내에서 전환

### 4-6. 공통 컴포넌트

| 기능 | 세부 요구사항 | 관련 스토리 |
|---|---|---|
| 헤더/푸터 | 전 페이지 공통. 태블릿·데스크톱 헤더는 비로그인 시 로그인/회원가입 버튼, 로그인 후에는 같은 자리에 아이디(사용자명) + 로그아웃 버튼을 노출 | US-15 |
| 반응형 네비게이션 | 모바일은 헤더 아래로 펼쳐지는 드롭다운(아코디언) 메뉴로 전환되며 네비게이션 항목(기업소개/메뉴소개/고객센터)만 노출하고 로그인/회원가입 링크는 포함하지 않음, 태블릿·데스크톱은 상단에 네비게이션과 로그인 상태 영역을 함께 가로로 노출 | US-16 |
| 관리자 모드 토글 | 관리자로 로그인해도 각 화면의 등록/답변 버튼은 평소 숨겨져 있으며, 헤더의 별도 아이콘/토글로 "관리자 모드"를 켰을 때만 노출됨 | US-18 |

### 5-1. 반응형 브레이크포인트

| 구분 | 화면 폭 | 주요 레이아웃 특징 |

| 태블릿 | 768px ~ 1023px | 2~3열 그리드, 헤더는 전체 메뉴 가로 노출, 메뉴소개 사이드바는 좌측 고정 |
| 데스크톱 | 1024px ~ 1440px | 3~4열 그리드, 콘텐츠 영역 최대 폭 고정 후 중앙 정렬, 여백을 넓게 사용 |

각 화면(메인페이지, 로그인/회원가입, 기업소개, 메뉴소개, 고객센터)의 브레이크포인트별 세부 동작은 4장의 화면별 "반응형" 항목을 따른다.
